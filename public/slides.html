Introduction

C'est quoi le Dynamic Programming ?

— Une méthode d'optimisation qui décompose un problème complexe en sous-problèmes plus simples
— On résout chaque sous-problème une seule fois et on stocke le résultat pour éviter les calculs redondants
— Inventé par Richard Bellman dans les années 1950 — le nom n'a rien à voir avec la "programmation" au sens code
Concepts fondamentaux

Deux propriétés essentielles

Sous-structure optimale

La solution optimale du problème contient les solutions optimales de ses sous-problèmes.

// Fibonacci F(n) = F(n-1) + F(n-2) ↑ ↑ sous-problèmes optimaux

Chevauchement des sous-problèmes

Les mêmes sous-problèmes sont résolus plusieurs fois dans l'approche naïve.

F(5) / \ F(4) F(3) ← déjà calculé ! / \ / \ F(3) F(2) F(2) F(1)
Approches

Top-Down vs Bottom-Up

Mémoïsation

Récursion + cache. On part du problème principal et on descend.

def fib(n, memo={}): if n in memo: return memo[n] if n <= 1: return n memo[n] = fib(n-1) + fib(n-2) return memo[n]

Tabulation

Itératif + tableau. On part des cas de base et on remonte.

def fib(n): dp = [0] * (n + 1) dp[1] = 1 for i in range(2, n + 1): dp[i] = dp[i-1] + dp[i-2] return dp[n]
Méthodologie

Résoudre un problème DP en 4 étapes

1 Identifier — Le problème a-t-il une sous-structure optimale et des sous-problèmes chevauchants ?
2 Définir l'état — Que représente dp[i] ou dp[i][j] ? C'est la question clé.
3 Écrire la récurrence — Comment dp[i] dépend des états précédents ? Le cœur de l'algorithme.
4 Cas de base — Initialiser les valeurs connues : dp[0], dp[1], première ligne/colonne...

En pratique

Quand utiliser le DP ?

✓ Le problème demande un optimum (min, max, nombre de façons)
✓ On peut le décomposer en sous-problèmes qui se répètent
✓ La solution récursive a une complexité exponentielle
✗ Le problème n'a pas de sous-structure optimale (ex: plus long chemin dans un graphe général)
✗ Les sous-problèmes sont tous distincts — pas de chevauchement, pas de gain
